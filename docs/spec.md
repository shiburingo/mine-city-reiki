# 仕様

## 目的
- 地方自治法と美祢市例規を横断参照する。
- 他アプリから必要条文を検索・参照できるようにする。
- 自然文質問に対して関連条文候補を提示する。

## データ構造
- `law_documents`: 文書単位
- `law_articles`: 条文単位
- `law_search_terms`: 日本語検索用の転置インデックス
- `law_synonyms`: 同義語辞書
- `sync_settings`: 月次更新設定
- `sync_runs`: 同期履歴
- `search_query_cache`: 検索結果キャッシュ
- `ask_query_cache`: 質問応答キャッシュ

## 検索
- 日本語形態素解析とキーワード抽出で `law_search_terms` を生成する。
- 検索時は転置インデックスから候補条文を高速抽出し、その後アプリ側で再スコアリングする。
- 検索語は `law_synonyms` を使って拡張する。例: `例規 -> 条例 / 規則 / 要綱`。
- 同一条件の再検索は `search_query_cache` を返し、DB負荷と応答時間を下げる。
- 同期更新時は `sync_settings.cache_generation` を更新し、旧キャッシュを無効化する。
- インデックス未整備時のみ旧来の LIKE 検索へフォールバックする。
- 回答は候補提示型であり、法的判断を断定しない。

## 質問応答
- `ask` は検索結果上位候補を使って、関連文書と条文抜粋を返す。
- 同一質問は `ask_query_cache` を返し、短時間の再実行を高速化する。
- 回答生成は要約や断定ではなく、条文候補提示を優先する。

## 月次更新
- DB に日・時・分・対象ソースを保存する。
- サーバー側では `run_due_sync.py` を定期実行し、設定時刻を過ぎた月だけ同期する。

## 他アプリ参照 API
- `reference/search` はキーワード検索・通称検索の入口です。
- `reference/document/:id` は他アプリから文書/条文詳細へリンクするための取得口です。
- `ask` は質問文から候補条文を返します。法的判断の自動確定は行いません。
- 同一ドメイン配下の業務アプリから相対パスで利用する前提です。
